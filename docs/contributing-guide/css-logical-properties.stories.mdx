# CSS Logical Properties
The Infermedica component library uses [CSS logical properties](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Logical_Properties)
that provide the ability to control the layout through logical, not physical representations of directions and dimensions.
It give us a lot more power to control our websites with minimal style changes, no matter what type of language we use
English, Arabic, etc.

<PageOutline/>

## How to use
You can use the following `use-logical` mixin, which transforms conventional ltr CSS physical value to CSS logical properties.

<Code code={`@include mixins.use-logical(element, property, value);`}/>

### Mixin arguments:
#### Property
- margin
- border
- inset
- border-width
- border-style
- border-color
- border-radius

#### Value
Please use conventional ltr css value for property (top right bottom left) e.g. `20px 30px 10px 40px`:
<Code code={`
{property}-block-start: 20px;
{property}-block-end: 10px;
{property}-inline-start: 30px;
{property}-inline-end: 40px;
`}/>

- When a value has three declaration, it is a shortcut to set `inline-end` property e.g `20px 30px 10px`

<Code code={`
{property}-block-start: 20px;
{property}-block-end: 10px;
{property}-inline-start: 30px;
{property}-inline-end: 30px;
`}/>

- When a value has two declaration, it is a shortcut to set block and inline properties e.g `20px 30px`

<Code code={`
{property}-block-start: 20px;
{property}-block-end: 20px;
{property}-inline-start: 30px;
{property}-inline-end: 30px;
`}/>

- When a value has one declaration, it is a shortcut to set all properties e.g. `20px`

<Code code={`
{property}-block-start: 20px;
{property}-block-end: 20px;
{property}-inline-start: 20px;
{property}-inline-end: 20px;
`}/>

### How it works in practice
- sass input
<Code code={`
  @include mixins.use-logical(button, padding, 12px 32px 8px);
`}/>

- css output
<Code code={`
padding-block:
  var(--button-padding-block-start, var(--button-padding-block, var(--button-padding-logical, 12px)))
  var(--button-padding-block-end, var(--button-padding-block, var(--button-padding-logical, 8px)));
padding-inline:
  var(--button-padding-inline-start, var(--button-padding-inline, var(--button-padding-logical, 32px)))
  var(--button-padding-inline-end, var(--button-padding-inline, var(--button-padding-logical, 32px)));
`}/>

## Easy way for rtl support
CSS logical properties respond automatically to text direction and typing mode. This means that the left and right sides will be automatically reversed and
you won't need to add styles for rtl layouts anymore.

<Code code={`.ui-list {
  // deprecated
  margin: 0 var(--space-4) 0 var(--space-8);'\n
  [dir="rtl"] & {
    margin: 0 var(--space-8) 0 var(--space-4);
  }
  // Please, use this instead
  @include mixins.use-logical(list, margin, 0 var(--space-4) 0 var(--space-8));
}
`}/>
